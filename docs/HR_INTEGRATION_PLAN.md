# 員工與排班與打卡 — 三模組整合方案

## 現狀問題

| 模組 | 頁面 | 問題 |
|------|------|------|
| 排班管理 | SchedulesPage | 用自訂時間（09:00-18:00）而非固定早/晚更；日曆上員工無顏色區分 |
| 打卡系統 | AttendancePage | 獨立運作，未與排班綁定打卡資格 |
| 員工與薪酬 | PayrollPage | 員工資料管理與排班/打卡無資料連動 |

## 關鍵資料關聯

```
employees (master)
   │
   ▼
schedules (排班表)
   ├── AM shift: start_time='09:00', end_time='14:00'
   ├── PM shift: start_time='14:00', end_time='19:00'
   └── shift_type: 'morning' | 'evening'
   │
   ▼ (驅動)
attendance_eligibility (打卡資格)
   │
   ▼ (產生)
attendance (打卡記錄)  ←  employee只能打卡當日有排班的時段
   │
   ▼ (計算)
salary_records (薪資明細)
```

## 排班→打卡 核心邏輯

```typescript
// 打卡資格判斷
function canClockIn(employeeId: string, date: string): boolean {
  const schedule = getSchedule(employeeId, date);
  // 有排班且狀態為 confirmed/scheduled 才可打卡
  return schedule?.status === 'confirmed' || schedule?.status === 'scheduled';
}

// 決定員工當日時段 (AM/PM/全天)
function getShiftType(startTime: string, endTime: string): string {
  if (startTime >= '09:00' && startTime < '12:00') return 'morning';
  if (startTime >= '12:00' && startTime < '18:00') return 'evening';
  return 'full_day';
}
```
